Migrating from JW5 to JW6

This guide enumerates the differences in functionality for JW Player 6 (JW6) compared to JW Player 5 (JW5). It is written for existing JW Player users, to assist them in upgrading from JW5 to JW6.

For each topic, both new/enhanced and removed/descoped features are listed. In the latter case, suggestions for alternative setups are provided.

Media Support

Since JW Player relies upon the underlying Flash and HTML5 frameworks for media playback, not a whole lot has changed in terms of media support.

New or Enhanced

Defined list of formats
For JW6, we took an effort to clearly define all supported media formats, including their expected extensions and mimetypes. JW6 officially supports MP4/FLV/WebM video, AAC/MP3/Vorbis audio, YouTube streams and RTMP streams.
Setting multiple formats
Setting multiple formats for one clip (RTMP+MP4, MP4+WebM, etc) has become more flexible and easier in JW6. Simply provide a list of sources using inline playlists or mRSS feeds. JW Player automatically detects and filters which formats can be played on each device.

Removed or Descoped

Playback of JPG/PNG/GIF images
JW5 supported playback of JPG/PNG/GIF images in Flash mode (not in HTML5). Since JW Player focuses on audio/video playback, we removed image support in JW6. We suggest using one of the many free (and much better suited) JavaScript image galleries for playing images.
Playback of Ogg videos
Contrary to JW5, JW6 does not officially support the Ogg video format anymore. WebM video is more modern, more widely used, and supported by roughly the same percentage of the browser market.

Device Support

One of the major internal changed in JW6 is a full refactor of its HTML5 playback. The HTML5 rendering mode is now leaner, more stable and in support of recent browser features.

New or Enhanced

Defaulting to HTML5
JW6 automatically detects if a device supports Flash and HTML5. It also detects if the embedded media format can be played back in Flash or HTML5. If both are possible, JW6 defaults to HTML5, but you can override this using the primary configuration option.
HTML5 Fullscreen Support
JW6 supports true fullscreen playback in HTML5 mode using the HTML5 Fullscreen API. This is supported in Firefox, Chrome and Safari today.
Download fallback
If a device supports neither Flash nor HTML5, JW6 can render a so-called download fallback: a nicely formatted download link to the media that is picked up by a built-in media player. This works great on e.g. Blackberry or Android 2.2 or older. We clearly defined which formats this fallback applies to, plus added an option to disable it.

Removed or Descoped

Flash mode on Android
Since Flash is performing suboptimal on Android devices, we already defaulted to HTML5 for media playback on Android. In JW6, Flash on Android support is removed altogether. We encourage you to make your media compatible with HTML5 when targeting Android, also since Android 4+ does not include Flash anymore.

Embedding

For JW6, we rationalized the various methods used to embed JW Player.

New or Enhanced

Separate embedder
In JW6, the library that manages the player embed (jwplayer.js) is separated from the actual HTML5 player (jwplayer.html5.js). This means the full HTML5 player is not loaded if the Flash player (jwplayer.flash.swf) is selected, saving ~100kb of data transfer.
Setup and media errors
JW6 features a documented list of player setup and media playback errors. All errors are descriptive (no more "Error #2032") and clearly printed in the player space. See Troubleshooting Your Setup to identify and resolve an error.

Removed or Descoped

Embedding with <embed> or SWFObject
JW6 focuses on HTML5/mobile and on robust player-page interaction using the Javascripting API. Embedding only the SWF player is not supported anymore. If you rely heavily on SWF-only embedding, we suggest you continue using JW5 (which we support until JW7 launches).
Embedding over a <video> tag
JW6 supports many functionalities not supported by a <video> tag (e.g. Flash, RSS, YouTube, RTMP, VAST). Additionally, longer term promises of an inline <video> element (progressive enhancement, video SEO, accessibility) are not there yet. Therefore, JW6 does not support the enhancing of a video element yet.

Configuration

New or Enhanced

The primary option instead of the modes block
The Flash/HTML5 mode selection in JW6 is greatly enhanced compared to JW5. Therefore, the modes configuration block is not needed anymore. Instead, you can simply set your preferred rendering mode with the primary option. JW Player will figure out which devices support which modes and formats.

Removed or Descoped

The flashplayer option
In JW6, it is not possible (not needed) to specifically point to the location of jwplayer.flash.swf. Instead, JW6 presumes the file (and jwplayer.html5.js) is in the same location as the main jwplayer.js library.
The link and displayclick options
These options are removed to tighten up display UX and avoid XSS security issues. Publishers can still implement Call-To-Action features in JW6 using the logo block, or respond to display clicks using the onDisplayClick API call.
The config XML options
The removal of <embed> support also removed the need for external XML configurations. They can now be loaded as part of the player itself, e.g. using JSONp callbacks or an <iframe> tag.

Playlists

New or Enhanced

Removed or Descoped

Feeds in XSPF/ASX/ATOM format
Media playback has standardized around Media RSS in recent years. In order to improve clarity around documentation and reduce the filesize of both Flash and HTML5 player, we removed support for other feed formats. Since mRSS supports any feature supported in another format, migrating to mRSS is not a technical problem.

Skinning

New or Enhanced

New default skin
A much slicker and smoother default player skin replaces the (somewhat dated) design of JW5.
Transition animations
The skinning model is enhanced with subtle animations for e.g. button rollovers, controlbar hiding and media transitions.

Removed or Descoped

SWF skinning
The PNG/XML skinning model of JW5 is more flexible, less error-prone and working in HTML5 than the SWF skinning model of JW4. Therefore, SWF skinning has been removed altogether from JW6.

Streaming

New or Enhanced

Support for multiple qualities
Built-in support eases the setup of closed captions and multiple quality levels. Selecting a language or quality is done through sleek, new overlay menus. JavaScript APIs are available to interact with captions and levels.

Removed or Descoped

RTMP tunneling fallback
Due to the diversity of RTMP setups (ports, firewalls, protocols), the automated RTMP tunneling feature in JW5 tended to create as many issues as it solved. Therefore, this is removed from JW6. Instead, we ensure the onError() event is called when an RTMP server cannot be reached. Publishers can then fallback from RTMP in any way they wish using a few lines of JavaScript code.
