Embedding the Player

This guide explains how to embed JW Player into your site, load the media file(s) you want to play and tweak the player's layout and playback behavior.

Player Setup

Since JW Player is CDN hosted, there is no need to install and upload the player to your site. Therefore, embedding the JW Player is a simple, 3-step process. Open your web page in an HTML editor and:

  1. Insert the JW Player script from our CDN into the head of your page: 
    <script type="text/javascript" src="http://jwpcdn.com/player.js" ></script>
  2. Insert a <div> element with a unique id at the location you want the player to appear: 
    <div id="myElement">Loading the player ...</div>
  3. Insert a script that invokes the JW Player setup() call on your <div>:
    4. <script type="text/javascript">
    jwplayer("myElement").setup({
        file: "/uploads/example.mp4",
        height: 360,
        image: "/uploads/example.jpg",
        width: 640
    });
</script>

The last code block is where the heavy lifting happens. The script in it tells the JW Player library to setup a player instance over the DOM element with ID container. Inside the setup call, the script provides a list of options for configuring the player. In this setup, only the file, height, image and width options are set, but many more are available. They are listed below.

Configuration Options

There are three options for configuring which media file(s) to load in the player:

file
URL to a single MP4/FLV/WebM video, AAC/MP3/Vorbis audio, YouTube page or HLS/RTMP manifest to play. See Supported Media Formats for an overview. Use the playlist option for more advanced file setups.
image
URL to a JPG/PNG poster image to display before playback is started (and during playback for audio). If not set, the image of the first item in the playlist is used.
playlist
Array with playlist data or URL to RSS feed. Enables all sophisticated media playback options, like multiple formats, multiple quality levels or multiple items. See Working with Playlists and Loading RSS Feeds for more info.

Overall, the file option can be used only for basic setups, like playing a single MP4 video or MP3 song. Use the playlist option for everything else, like an MP4+WebM source, SD+HD quality or a list with multiple items. See Working with Playlists for more info.

Layout

These are the options for configuring the layout of the player:

controlbar
Whether to display the video controlbar at the bottom of the screen. Can be false or true (default).
height
Height of the player in pixels. We recommend setting this to at least 180, so all UI elements will fit. However, it can be set to a small size (e.g. 40) for audio-only playback. Defaults to 360.
icons
Whether to display the Play and Buffer icons in the center of the screen. Can be false or true (default).
skin
Which skin to use for styling the player. Can be the identifier of a CDN hosted skin or the URL to a self-hosted skin. See Using JW Player Skins for more info.
stretching
How to resize the poster and video to fit the display. Can be none (keep original dimensions), exactfit (stretch disproportionally), uniform (stretch proportionally; black borders) or fill (stretch proportionally; parts cut off). Defaults to uniform.
width
Width of the player in pixels. We recommend setting this to at least 320. On smaller players, certain UI elements may not fit. Defaults to 640.

Behavior

And these are the options for configuring the setup and playback behavior:

autostart
Automatically start playing the video on page load. Can be true or false (default). Autostart does not work on mobile devices like iOS or Android.
fallback
Whether to render a nice download link for the video if HTML5 and/or Flash are not supported. Can be true (a fallback is rendered; the default) or false (the original HTML is not touched).
mute
Whether to have the sound muted on startup or not. Can be false (default) or true.
primary
Which rendering mode to try first for rendering the player. Can be html5 (default) or flash.
repeat
What to do when an item finished playback. Can be list (play the full playlist once), none (stop playback after every item), always (repeat the playlist forever) or single (repeat the active item forever). Defaults to list.
startparam
Querystring parameter for seeking to not-yet-download portions of FLV or MP4 files in Flash mode. Not needed for HTML5 (works by default). See Pseudostreaming in Flash for more info.

Advanced Options Blocks

In addition to above configuration options, a list of configuration blocks is available to enable the more advanced functionalities of JW Player. For example, the sidebar block can be used to display the playlist in a sidebar next to the video display:

jwplayer("myElement").setup({
    playlist: "http://example.com/playlist.rss",
    height: 360,
    sidebar: {
        position: 'right',
        size: 320
    },
    width: 960
});

As you can see, the sidebar is an object that itself contains a set of configuration options. In this case, the sidebar is setup to the right of the display and set 320 pixels wide. See the Working with Playlists guide for more info on setting up the sidebar.

The following advanced option blocks are available:

sidebar
Config block for displaying a sidebar with the Playlist contents.
captions
Config block for displaying captions or subtitles.
logo
Config block for displaying an on-screen watermark.
rtmp
Config block for tweaking the playback of RTMP streams.
gapro
Config block for pinging events to Google Analytics.
related
Config block for displaying an overlay with Related Videos.
sharing
Config block for displaying an overlay with Social Sharing links.
ova
Config block for loading ads using the VAST/VPaid standards.
adtonomy
Config block for loading ads using self-hosted assets.

Player Removal

In addition to setting up a player, the JW Player embed script contains a function to unload a player. It's very simple:

jwplayer("container").remove();

This formal remove() function will make sure the player stops its streams, the DOM is re-set to its original state and all event listeners are cleaned up.
