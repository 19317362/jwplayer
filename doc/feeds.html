Loading RSS Feeds

RSS feeds can be used to load playlists with multiple pieces of content into JW Player, but also to load multiple media formats and/or multiple quality levels for each piece of content.

JW Player requires the RSS feed to be valid and using the Media RSS extension to define content and thumbnail. Any playlist attributes not supported by RSS and Media RSS can be added using the JWPlayer namespace.

Introduction

JW Player supports two mechanisms for loading playlists:

  • A inline configuration. It is enabled by setting the playlist configuration option to an array of playlist items.
  • An external RSS Feed. It is enabled by setting the playlist configuration option to the URL of an RSS feed.

See Working with Playlists for info on loading playlists inline, plus info on the structure of playlists in JW Player.

If you have the choice, inline configurations are preferred. They allow JW Player to load content faster (no extra file to load), plus they avoid any crossdomain loading issues.

A crossdomain loading issue appears if your player is embedded at http://somesite.com and your RSS feed is loaded from http://othersite.com. By default, this is not allowed by any browser, in both Flash and HTML5 mode. See Crossdomain File Loading for more info and workarounds.

Example

Here's an example jwplayer().setup() block that configures a player to load an RSS feed:

<script type="text/javascript">
    jwplayer("container").setup({
        playlist: "http://example.com/playlist.rss"
    });
</script>

And here's the contents of an example RSS feed. It defines 2 playlist items:

<rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" > 
  <channel> 
    <title>Example RSS Playlist</title> 
    <item> 
      <title>Sintel</title>
      <description>Sintel is a fantasy computer generated short movie. It's the third 
         release from the Blender Open Movie Project.</description>
      <media:thumbnail url="http://example.com/thumbs/sintel.jpg" />
      <media:content url="http://example.com/videos/sintel.mp4" /> 
    </item>
    <item>
      <title>Big Buck Bunny</title>
      <description>Big Buck Bunny is a short animated film by the Blender Institute,
         part of the Blender Foundation.</description>
      <media:thumbnail url="http://example.com/thumbs/bunny.jpg" />
      <media:content url="http://example.com/videos/bunny-sd.mp4" />
    </item> 
  </channel> 
</rss>

When authoring RSS feeds, do not forget to add the xmlns:media namespace to the top XML element. Otherwise, JW Player won't be able to parse the feed.

Mapping

The full set of item and source properties that can be set for inline playlists is also supported for RSS feeds. Here's how all item properties map to Media RSS elements:

JW PlayerRSS FeedComments
titletitle
descriptiondescription
mediaidguidNeeds isPermalink="false" attribute
imagemedia:thumbnail@url
durationmedia:content@durationNeeds to be a number

All source properties are actually mapped to atributes of the media:content RSS element:

JW PlayerRSS FeedComments
filemedia:content@url
typemedia:content@typeShould be a mimetype (video/mp4)
widthmedia:content@width
labelmedia:content@heightamended with a p

Having multiple media:content elements per RSS item is allowed, so the multi-format and multi-quality options of JW Player can be achieved with RSS as well. Here's an example feed with one item that has an SD + HD quality video:

<rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" > 
  <channel> 
    <title>Example RSS Playlist</title> 
    <item> 
      <title>Sintel</title>
      <description>Sintel is a fantasy computer generated short movie. It's the third 
         release from the Blender Open Movie Project.</description>
      <media:thumbnail url="http://example.com/thumbs/sintel.jpg" />
      <media:content url="http://example.com/videos/sintel.mp4" 
         duration="50" height="360" type="video/mp4" width="640" />
      <media:content url="http://example.com/videos/sintel-hd.mp4" 
         duration="50" height="720" type="video/mp4" width="1280" /> 
    </item>
  </channel> 
</rss>

JWPlayer Namespace

For certain advanced features, like Closed Captioning or Related Videos, additional properties can be set per playlist item. RSS does not support these out of the box, but you can add these properties by using the JW Player namespace. It works exactly like the Media namespace, with an xmlns declaration at the top and a jwplayer: prefix on the elements.

Here's an example setting the streamer option that's needed to play a single RTMP stream:

<rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" 
  xmlns:jwplayer="http://developer.longtailvideo.com/"> 
  <channel> 
    <title>Example RSS Playlist</title> 
    <item> 
      <title>Sintel</title>
      <description>Sintel is a fantasy computer generated short movie. It's the third 
         release from the Blender Open Movie Project.</description>
      <media:thumbnail url="http://example.com/thumbs/sintel.jpg" />
      <media:content url="sintel.mp4" />
      <jwplayer:streamer>rtmp://example.com/vod</jwplayer:streamer>
    </item>
  </channel> 
</rss>

Again, don't forget the top-level xmlns tags. Without them, JW Player will not be able to parse these feeds.
