Working with Playlists

The playlist is a powerful concept in JW Player that can be used to load multiple pieces of content into a single player. It can also be used to load multiple media formats and/or multiple quality levels for each piece of content. The playlist can be displayed as a sidebar to the video window.

The playlist is defined as part of the jwplayer().setup() call. See Embedding JW Player for more info on embedding the player. Instead of an inline, the playlist can also be defined in an external RSS feed. See Loading RSS Feeds for more info.

Playlist Structure

The playlist consists of two layers: item and source. An item contains all data for a distinct piece of content (e.g. its title) and a source contains all data for a distinct media resource (e.g. its file). The playlist contains one or more items, and each item contains one or more sources.

Item

The following item properties are recognized and used by JW Player:

title
Title of the item. It is displayed inside the play icon before playback starts, plus in the sidebar.
description
Short description of the item. It is displayed in the sidebar.
mediaid
Unique identifier of this item. Used by advertising, analytics and discovery services.
image
Poster image representing the item. Displayed below the play icon before playback starts, plus in the sidebar.
duration
Duration of the item in seconds. It is displayed in the sidebar. It cannot be used to restrict the duration of the item.
sources
Array with one or more media sources for this item. Is required. See below.

For certain advanced features, like Closed Captioning, RTMP Streaming or Related Videos, additional properties can be set per playlist item. See the respective guides for more info.

Source

The following source properties are recognized and used by JW Player:

file
URL of the source. Is absolutely required.
width
Width of the source in pixels. Used for quality selection heuristics. Used when setting multiple qualities of the same format.
label
Human-readable description of the source. Displayed in the quality selection popup. Used when setting multiple qualities of the same format.
type
Media type of the source. Can be mp4, flv, webm, aac, mp3, vorbis, hls, rtmp or youtube. Is only needed when the file property does not contain a recognized file extensions (like .mp4 or .m4v for mp4). See Supported Media Formats for more info.

Playlist Examples

Let's take a look at a few examples of playlists. These examples only show the contents of the setup() code block. See Embedding JW Player on how to set the script library and container element.

Setting Multiple Formats

This setup has one item in the playlist with two sources: MP4 and WebM. JW Player will automatically display the MP4 source on Chrome/IE/Safari and the WebM source on Firefox:

<script type="text/javascript">
    jwplayer("container").setup({
        playlist: [{
            image: "/assets/sintel.jpg",
            sources: [{
                file: "/assets/sintel.mp4",
            },{
                file: "/assets/sintel.webm"
            }],
            title: "Sintel Movie Trailer"
        }
    });
</script>

See Browser & Device Support for more info on browser support and mode/source selection.

Setting Multiple Qualities

This setup has one item in the playlist with two sources. Both are MP4, but one is 640x360 pixels (SD) and the other is 1280x720 pixels (HD):

<script type="text/javascript">
    jwplayer("container").setup({
        playlist: [{
            image: "/assets/sintel.jpg",
            sources: [{
                file: "/assets/sintel.mp4",
                label: "360p",
                width: 640
            },{
                file: "/assets/sintel-hd.mp4",
                label: "720p",
                width: 1280
            }],
            title: "Sintel Movie Trailer"
        }
    });
</script>

On desktop browsers, JW Player will now display a quality selection menu in the controlbar, allowing viewers to toggle between "360p" and "720p" quality.

On mobile devices, JW Player will select the 360p version for devices whose screen is closer to 640 pixels wide and the 720p version for devices whose screen is closer to 1280 pixels wide. This is a poor man's automated quality selection; see Using HLS Streaming for a better setup that also take bandwidth into account.

Setting Multiple Items

Finally, this setup contains two playlist items, each with a title, image and single media source (mp4):

<script type="text/javascript">
    jwplayer("container").setup({
        playlist: [{
            image: "/assets/sintel.jpg",
            sources: [{
                file: "/assets/sintel.mp4"
            }],
            title: "Sintel Movie Trailer"
        },{
            image: "/assets/bunny.jpg",
            sources: [{
                file: "/assets/bunny.mp4"
            }],
            title: "Big Buck Bunny Movie Trailer"
        }]
    });
</script>

Combining multiple media formats, multiple quality levels and multiple playlist items is no problem. For example, one could setup a playlist with 5 items that each contain an RTMP manifest, an HLS manifest, an SD/HD MP4 file and an SD/HD WebM file...

Displaying the Sidebar

The playlist can be displayed as a sidebar to the video window, using the sidebar configuration block. This allows users to scroll through it and play specific items. The configuration block has the following properties:

layout (basic)
Layout of the playlist items in the sidebar. Can be basic (title and duration) or extended (image, title, duration and 3 lines of description).
position (none)
Position of the sidebar relative to the video display. Can be bottom (below the display), none (no sidebar) or right (to the right of the display).
size (180)
Width (if position is right) or height (if position is bottom) of the sidebar. This is basically the amount of pixels the sidebar "steals" from the video window.

Here is an example setup with a sidebar that's positioned to the right of the video with an extended layout:

<script type="text/javascript">
    jwplayer("container").setup({
        playlist: "http://example.com/playlist.rss",
        height: 360,
        sidebar: {
            layout: 'extended',
            position: 'right',
            size: 320
        },
        width: 960
    });
</script>
